# Campaign Design

## Overview
This document defines how the phishing simulation campaign will be structured, executed, and monitored within a controlled and ethical environment.

## Campaign Objective
- Evaluate user awareness against phishing attacks.
- Measure interaction metrics such as email opens, clicks, and submissions.
- Demonstrate phishing techniques for educational purposes.

## Target Audience
- Test users or consent-based participants.
- Non-production accounts only.
- Controlled user group defined for simulation.

## Phishing Scenario
- Email impersonating a trusted service or organizational notification.
- Message designed using social engineering techniques (urgency, authority, curiosity).
- Embedded link leading to a simulated landing page.

## Email Design
- Professional-looking template.
- Clear call-to-action button or link.
- Tracking enabled for interaction metrics.

## Landing Page Design
- Simulated login or information submission page.
- Visual similarity to a trusted service (educational simulation only).
- Data capture limited to simulated or dummy inputs.

## Infrastructure Setup
- Domain or subdomain configured for simulation.
- SMTP service for controlle
