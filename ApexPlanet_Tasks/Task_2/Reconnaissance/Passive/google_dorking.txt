Metasploit Exploit modules are components within the Metasploit Framework used to exploit known vulnerabilities in target systems.  These modules enable security professionals and researchers to test system vulnerabilities, assess impact, and develop custom payloads for further analysis. 

Active Exploits: Exploit a specific host, run to completion, and exit. Example: msf exploit(ms08_067_netapi) targets a Windows SMB vulnerability to gain remote access. 
Passive Exploits: Wait for incoming connections and exploit clients (e.g., web browsers). Example: msf exploit(windows/browser/ani_loadimage_chunksize) triggers a stack overflow when a user visits a malicious webpage. 
Common Exploits:
EternalBlue (ms17_010_eternalblue): A reliable remote exploit for Windows systems with unpatched SMB services, granting SYSTEM-level access.
vsftpd 2.3.4 Backdoor: Exploits a known vulnerability in vsftpd to gain a remote shell on Metasploitable 2.
Cisco RV320/RV325 RCE (linux/http/cisco_rv32x_rce): Uses two vulnerabilities to achieve unauthenticated remote code execution on routers via HTTPS.
Module Structure: Exploit modules follow a standard Ruby class format, including Name, Description, Targets, Payload, References, and check/exploit methods. Example: sample exploit. 
Usage Workflow:
Use use exploit/<module_name>
Set required options: RHOST, LHOST, PAYLOAD
Run exploit or run to initiate the attack.
Interact with sessions using sessions -i <id>.
